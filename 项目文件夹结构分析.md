# 项目文件夹结构分析

> 小红书 AI 代理系统 - 三层架构目录说明

---

## 整体架构

```
self-study-agent/
├── automation/          # Python 自动化层（爬虫 + AI + 数据存储）
├── backend/             # Node.js API 层（Express 后端服务）
├── frontend/            # React 前端层（用户界面）
├── data/                # 数据目录（SQLite、截图、用户画像）
├── .env                 # 环境变量配置
└── README.md
```

---

## 一、automation/ - Python 自动化层

> 职责：浏览器自动化、AI 分析、数据存储

### 核心文件

| 文件 | 功能 | 作用 |
|---|---|---|
| `main.py` | 每日任务编排 | 入口文件，串联"浏览 → 分析 → 互动 → 存储"完整流程；支持 `--profile`（生成画像）、`--daily`（单次任务）、`--schedule`（定时调度）三种模式 |
| `browser.py` | 浏览器自动化 | 基于 DrissionPage 驱动浏览器：初始化、登录等待、采集发现页、进入详情页、点赞/收藏、截图、滚动、关闭详情页等 |
| `analyzer.py` | AI 分析引擎 | 集成 DeepSeek API：生成用户画像、兴趣判断（标题初判 + 详情二次判断）、API 调用封装（含指数退避重试） |
| `summarizer.py` | 笔记总结 | 对互动笔记生成结构化摘要（核心观点、关键要点、兴趣匹配原因、AI 标签） |
| `chroma_store.py` | 向量库存储 | ChromaDB 封装：写入笔记语义信息、向量检索、RAG 问答（非流式 + 流式） |
| `chroma_server.py` | Chroma HTTP 服务 | Flask 服务，暴露 `/query` SSE 接口，供 Node.js 后端代理调用 |
| `config.py` | 配置常量 | API 密钥、文件路径、服务端口、每日上限等配置 |
| `init_db.py` | 数据库初始化 | SQLite 表结构创建和迁移 |
| `requirements.txt` | Python 依赖 | DrissionPage、openai、chromadb、flask、schedule 等 |

### vendor/ 子目录

> Python 依赖的 vendor 化目录，避免 pip 安装，便于项目迁移

包含 chromadb、openai、flask、httpx、numpy 等第三方库的完整源码。

## 二、backend/ - Node.js API 层

> 职责：RESTful API、SSE 流式代理、数据库操作

```
backend/
├── src/
│   ├── index.ts          # Express 服务入口
│   ├── db/
│   │   └── sqlite.ts     # SQLite 数据库操作封装
│   └── routes/
│       ├── notes.ts      # 笔记相关 API
│       └── chat.ts       # AI 对话 SSE 代理
├── package.json
└── tsconfig.json
```

### 核心文件

| 文件 | 功能 | 作用 |
|---|---|---|
| `src/index.ts` | Express 入口 | 创建 Express 实例，注册路由、中间件（CORS、JSON 解析、静态文件服务），启动 HTTP 服务 |
| `src/db/sqlite.ts` | 数据库操作 | SQLite 连接管理、表结构定义、笔记 CRUD 查询（分页、按标签筛选、详情查询） |
| `src/routes/notes.ts` | 笔记 API | `GET /api/notes`（分页列表）、`GET /api/notes/:id`（详情）、`GET /api/tags`（标签列表） |
| `src/routes/chat.ts` | 对话代理 | `POST /api/chat` SSE 接口：接收前端问题，通过 `http.request` 转发到 Python Chroma 服务，逐 chunk 透传 SSE 响应 |

## 三、frontend/ - React 前端层

> 职责：用户界面、笔记展示、AI 对话交互

```
frontend/
├── src/
│   ├── components/       # React 组件
│   │   ├── NoteLibrary.tsx      # 笔记库主组件（双列卡片 + 无限滚动）
│   │   ├── NoteCard.tsx         # 笔记卡片组件
│   │   ├── NoteDetailModal.tsx  # 笔记详情弹窗
│   │   ├── NoteList.tsx         # 笔记列表容器
│   │   ├── ChatWindow.tsx       # AI 对话窗口
│   │   ├── ChatPanel.tsx        # 对话面板
│   │   └── Navbar.tsx           # 顶部导航栏
│   ├── App.tsx           # 根组件（双栏布局）
│   ├── App.css           # 全局样式（双栏布局、响应式）
│   ├── api.ts            # 前端 API 封装（axios + fetch SSE）
│   ├── types.ts          # TypeScript 类型定义
│   ├── main.tsx          # React 入口
│   └── index.css         # 基础样式
├── vite.config.ts        # Vite 配置（含代理）
└── package.json
```

### 核心文件

| 文件 | 功能 | 作用 |
|---|---|---|
| `App.tsx` | 根组件 | 双栏布局容器，左侧 `NoteLibrary`，右侧 `ChatWindow` |
| `App.css` | 全局样式 | 双栏 Flex 布局、响应式适配（小屏幕切换上下布局）、卡片样式、弹窗动画 |
| `api.ts` | API 封装 | `axios` 封装笔记列表/详情请求；`fetch` + `ReadableStream` 封装 SSE 流式对话 |
| `types.ts` | 类型定义 | `Note`（笔记）、`ChatMessage`（对话消息）等 TypeScript 接口 |
| `NoteLibrary.tsx` | 笔记库 | 双列瀑布流卡片布局，无限滚动加载，标签筛选，搜索 |
| `NoteCard.tsx` | 笔记卡片 | 展示封面图、标题、摘要、互动类型、兴趣评分 |
| `NoteDetailModal.tsx` | 详情弹窗 | 左右分栏：左侧图片轮播，右侧 Markdown 渲染的笔记详情 |
| `ChatWindow.tsx` | 对话窗口 | 消息列表、输入框、流式消息展示（打字机效果）、Markdown 渲染 |
| `vite.config.ts` | Vite 配置 | 开发代理：`/api` → `localhost:3001`，`/screenshots` → `localhost:3001` |

## 四、data/ - 数据目录

> 运行时生成的数据文件，不在版本控制中

| 文件/目录 | 内容 | 说明 |
|---|---|---|
| `data/notes.db` | SQLite 数据库 | 笔记元数据（标题、URL、摘要、标签、图片路径、互动类型等） |
| `data/screenshots/` | 截图文件 | 自动化过程中截取的笔记图片，前端通过 `/screenshots/filename` 访问 |
| `data/user_profile.json` | 用户画像 | AI 生成的用户兴趣偏好（类别、关键词、风格偏好、置信度） |
| `data/chroma/` | Chroma 向量库 | 笔记语义向量的持久化存储 |

## 七、启动依赖关系

```
Chroma 服务 (5002) ──必须先启动──▶ Express 后端 (3001)
                                      │
                                      ▼
                               React 前端 (5173)
```

| 服务 | 启动命令 | 端口 | 前置依赖 |
|---|---|---|---|
| Chroma 服务 | `python automation/chroma_server.py` | 5002 | 无 |
| Express 后端 | `cd backend && npm run dev` | 3001 | Chroma 服务 |
| React 前端 | `cd frontend && npm run dev` | 5173 | Express 后端 |
| 自动化任务 | `python automation/main.py --daily` | - | 无（独立运行） |

## 八、关键技术选型说明

| 技术 | 用途 | 选型理由 |
|---|---|---|
| DrissionPage | 浏览器自动化 | Python 生态中功能最完善的浏览器驱动库，支持 CDP 协议 |
| DeepSeek API | AI 分析 | 性价比高，支持 JSON 格式输出，适合结构化分析任务 |
| SQLite | 关系型数据存储 | 轻量零配置，单文件便于迁移，适合个人工具场景 |
| ChromaDB | 向量存储 | 本地运行，无需外部服务，与 Python 生态集成好 |
| Express | Web 框架 | 轻量灵活，中间件生态丰富，SSE 支持原生 |
| React 19 | UI 框架 | 最新版本，并发特性（useTransition 等）提升交互体验 |
| Vite | 构建工具 | 开发启动快，HMR 即时更新，代理配置简单 |
| TypeScript | 类型系统 | 前后端统一使用，减少运行时错误，提升可维护性 |
